## The Challenge
`lisnavi` is a mature B2B service, but because screens were being added on an ad-hoc basis, the following problems had emerged:

* Slight design inconsistencies in parts used across different Figma files
* Many near-duplicate UIs serving the same purpose
* A management structure for components that diverged from the implementation

## The Approach
After interviewing the engineering team, I rebuilt the Figma Library from scratch in line with the component philosophy of [MUI](https://mui.com/), the UI library they were using.

The library was created after carefully reading the implementation-side documentation, ensuring that any MUI component or implementation option you choose maps accurately to Figma.
